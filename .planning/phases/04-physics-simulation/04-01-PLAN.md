Class structure:
```csharp
public class PhysicsSimulator
{
    // Constants
    private const float FixedDt = 1f / 60f;  // 60Hz physics
    private const float MaxAccumulation = 5f * FixedDt;  // Prevent spiral of death
    private const float BoxHalfSize = 1.0f;  // Box is 2x2x2 centered at origin
    private const float ParticleRadius = 0.02f;  // Approximate particle size

    // State
    private float _accumulator = 0f;
    private bool _isRunning = true;  // PHYS-04: automatic mode

    // Parameters
    public float Bounciness { get; set; } = 1.0f;  // Perfectly elastic default
    public float BoxSize { get; set; } = 2.0f;  // Full box dimension

    // Methods
    public void Update(float deltaTime, ParticleData[] particles)
    public void StepSimulation(float dt, ParticleData[] particles)
    public void SetRunning(bool running)  // Enable/disable automatic mode
}
```

Fixed timestep implementation (per Pitfall 3):
1. Accumulate deltaTime: `_accumulator = Math.Min(_accumulator + deltaTime, MaxAccumulation)`
2. While accumulator >= FixedDt:
   - Call StepSimulation(FixedDt, particles)
   - Subtract FixedDt from accumulator

StepSimulation logic:
1. For each particle:
   - Update position: `pos += vel * dt`
   - Check boundaries for each axis:
     - If pos.X > BoxHalfSize - ParticleRadius:
       - pos.X = BoxHalfSize - ParticleRadius
       - vel.X *= -Bounciness
     - If pos.X < -BoxHalfSize + ParticleRadius:
       - pos.X = -BoxHalfSize + ParticleRadius
       - vel.X *= -Bounciness
   - Repeat for Y and Z axes

Box alignment:
- The rendered box uses vertices at +/- 1.0 (2x2x2 units)
- BoxHalfSize = 1.0 matches the rendered box
- ParticleRadius accounts for particle visual size

Thread safety:
- PhysicsSimulator is not thread-safe; all calls from main/UI thread
- No locking needed for this phase</action>
